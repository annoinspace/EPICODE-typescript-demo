TypeScript Intro Notes

General overview

TS is a super-set of Javascript, it helps us finds bugs before we run our code. It must be compiled before we can view in our browser. You must have knowledge of Javascript to be able to use TypeScript as it builds on top of it.

The TS compiler

The compiler configures a Javascript version of our TypeScript code by running tsc filename.ts in the terminal.
TS compiler helps to convert code written in typescript to javascript since the browser and html does not support or understand typescript code unless it being converted.

Type Inference

This is when no explicit type is provided, so an expression is written to help TS calculate a "best common type".
This essentially gives you some options as to what the type can be, instead of having only one.

The ‘any’ Type

Defining a variable as 'any' lets you assign any type to it.

e.g
let anyVar: any = "a string"
anyVar = 5
the 'any' type kind of defeats the purpose of TypeScript since we can change the type, but can be used when we need to use multiple types.

The Union Type

This is a great way for us to use multiple types. The union types can be signified with --> |
e.g
let year: string | number
--> year = 2021
--> year = "2021"
using the union type in this example, both string and number types are valid.

The Tuple Type

Tuples are used within arrays, when you want specific types to be present in specific spots. If you define these spots, they must be filled.
e.g
let arrTuple: [string, number] = ["a string, 5"]
arrTuple: ["a",4] <-- follows the correct order of types
In this array, we will always have a string as the first element, and number as the second

Interfaces in TS

Interfaces enforce a certain structure of an object, this furthers our attempts in keeping code clean.

e.g
(we define an interface first with interface)
interface PersonInterface {
age: number;
name: string }

(now using the interface interface)
let aneesah: PersonInterface = {
age: 23;
name: "Aneesah" }

Generics in TS

Generics are used in instances where you don't know what type something is going to be.
Usually < T > is used in this case, but the T can be replaced with anything else. The < T > acts as a placeholder.
